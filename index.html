Introduction

Bob the Builder is intended to be a general purpose build tool dedicated for Vala language based projects. We try to keep Bob as simple and intuitive as possible mantaining it's elastic and quite powerful design.

Bob is loosely inspired by Maven build tool for Java using only it's general idea of plugins based architecture.

Bob project structure

Every project that is using Bob as a build tool has to share the same meaningful structure with source code located under appropriate folders:

/src/library/vala
Place all the Vala code here for a library related source code.
/src/library/vapi
Holds all VAPI files referenced in library source code.
/target/library
Here will lay all the libraries binaries generated by Bob in process of code compilation.
/src/main/vala
Whole source code located here is treated as runtime executable binary related.
/src/main/vapi
Holds all VAPI files referenced in runtime executable binary source code.
/target/main
Bob will put all binaries executables here after process of code compilation or any other activity related.

Plugins and recipies

As stated earlier Bob design is created around a Plugin based approach, where each plugin is responsible for performing a dedicated action on a set of project source code files. As for now we were able to implement a minimal subset of plugins to conform basic and minimal requirement that is creating binary file out of set of provided Vala source code files:

collect-sources
Plugin aims to traverse recursively through project directory structure and collect a list of Vala source code files. Then the list is strored in project properties available for other plugins.
build
It's basically a refactored ValaCompiler source code tuned a little bit to fit Bob builder architecture. It uses Vala source code files found by collect-sources plugin and goes through Vala compiler procedure (generating C code, GIR bindings, VAPI files, etc).

While having in mind generic nature of all Bob plugins (plugin will work the same way no matter what code it's running against) there is an easy way one can configure their default behavior. For every Vala project managed by Bob you can provide a project configuration file called Project Recipe - recipe.bob. This is just a simple JSON file with very meaningful structure. Let's have a quick look at simple example project recipe file:

{ "project": { "name": "Test project", "shortName": "test", "version": "0.0.1" }, "plugins": { "collect-sources": { "verbose": true }, "build": { "verbose": false, "dependencies": [ { "dependency": "glib", "version": "2.0", "type": "pkg" }, { "dependency": "gobject", "version": "2.0", "type": "pkg" } ] } } }

Bob usage scenarios

Bob usage is rather straightforward and does not require and specialized terminal user skills. To run Bob with default settings naviage to project directory and type in system shell console:

$/project> bob

By default Bob will only print out the list of parsed plugins recipies and some version information as well:

[BobBuildRecipeParser] Parsing plugin recipe: collect-sources [BobBuildRecipeParser] Parsing plugin recipe: uncrustify [BobBuildRecipeParser] Parsing plugin recipe: build [BobBuildPluginsLoader] Loading plugins from directory: /opt/vala/bob-builder/target/plugins [BobBuilder] Bob the Builder, version = 0.0.1

To tell Bob to run any of available plugin just pass a list of plugins one as bob executable parameters:

$/project> bob collect-sources build

Questions and Answers

Why another build tool? Isn't make, Cmake, waf or any other out there enough?
Short answer is: nope. Every tool that already exists like make or Cmake isn't dedicated for Vala but rather for C/C++ code. Besides, configuration scheme for any of those build tools is greately complicated. Bob is here to make it easier and fit better for Vala.
I have an idea for a new possible plugin, how can I contribute?
If you are familiar with Vala (or Java or C#) go and for Bob builder source code repository on GitHub. If you don't know how do code - please post me a message or create a GitHub issue with short description, I could think of it in spare time.
